SyncMode  string

The different modes supported for fetching chat content during the initial call to BBMEnterprise.Messenger#fetchChatMessages.

Properties

Name Type Optional Description

NoSync

 

 

Indicates that an initial sync of a chat should not sync anything more than the number of messages requested. In this mode, all messages up to the latest received message will be marked as delivered, even though there may be messages before what was requested that had not been delivered yet.

SyncToDelivered

 

 

Indicates that an initial sync of a chat should sync at least as far back as the earliest message which had not yet been delivered. In this mode, calling BBMEnterprise.Messenger#chatMessageRead will mark all messages up to the specified message as read, even though there may be messages earlier than this that have not been delivered to any client.

SyncToRead

 

 

Indicates that an initial sync of a chat should sync at least as far back as the earliest message which had not yet been marked read by calling BBMEnterprise.Messenger#chatMessageRead.

Send an invitation to each user identified by their regId to participate in the chat identified the given chatId.

Parameters

Name Type Optional Description

string

 

The identifier for the chat for which invitations are to be issued. The local user must be a current participant in this chat to successfully invite others to join. The chat must not be a 1:1 chat, only conferences allow for the invitation of additional parties.

invitees

(string or Array of string)

 

This argument contains the regId(s) to which invitations to participate in the identified chat are to be sent. This argument will accept:

  • A regId when only a single user is to be invited.

  • An array of regIds when multiple users are being invited.

    Sending an invite to the regId of the local user is not supported and will result in the invite for this regId being rejected.
Throws

BBMEnterprise.Error.NotFoundError 

When the given chatId is not known to the SDK or the local participant is not a participant in the chat.

Returns

Promise containing (Promise or Array of Promise) 

When inviting a single user by passing a string as the invitees parameter, the promise returned will resolve to a single promise when the invitation has been sent.

When inviting multiple users by passing an array as the invitees parameter, the promise returned will resolve with an array of promises that will be resolved or rejected independently for each user invited to participate in the chat.

The promise may be rejected with any error. In this case, no invitations will have been sent.

chatMessageDestroy(chatId[, messageId]) → Promise

Destroy one or all messages previously sent by the local user in the given chat.

When a single messageId is specified, its contents are cleared locally, a message is sent instructing all participants to clear the message's contents, and the infrastructure removes the message from the chat.

When destroying all messages, the affected messages are removed locally, a message is sent instructing all participants to also remove all of the affected messages, and the infrastructure also removes all of the affected messages from the chat.

Synchronous errors are reported via an exception being thrown. Asynchronous errors are reported via the returned Promise being rejected.

Parameters

Name Type Optional Description

chatId

string

 

The identifier of the chat containing the message(s) to destroy.

messageId

BBMEnterprise.Messenger.MessageId

Yes

If specified, the message identifier of the single message to destroy. When omitted, all messages previously sent by the local user will be destroyed.
Throws

BBMEnterprise.Error.Error 

When a messageId is specified, but it is not suitable for recall. Only messages previously sent by the local user's identity may be recalled.

BBMEnterprise.Error.NotFoundError 

When the chat is not found, or when a messageId is specified but it is not found in the chat's local cache.

Returns

Promise 

A promise that will be resolved once the steps outlined above have completed successfully. The promise will be rejected if any steps fail to complete successfully.

chatMessageWatch(chatId, messageId, callback)

Register to receive notifications about a message. Any time the message is modified, the callback will be run to notify the interested party. If a message is known to the system, the callback will immediately be run with the message as it is currently known. If the message is not known to the system, then the callback will be called for the first time when it is.

Each callback function may be registered for more than one message, but may only be registered once per message.

Note that it is important to unwatch a message when you are done with it. The watcher will be tracked until it is explicitly unwatched, which may result in preformance degradation if care is not taken to unwatch for each watcher.

For an easier-to-use method which doesn't experience these resource leaks, this same information can be obtained from the events emitted by this class. However the events must be filtered by the caller to find desired events and the caller must rescan the entire message list after receiving a bulk message update event. As such, the performance of this API will be higher than could be achieved by listening to events.

    fetchChatMessages(chatId[, options]) → Promise containing any

    Retrieve the messages associated with a specified chat, including both messages that are cached locally, and possibly more retrieved from the server. This will initiate download of messages and resolve once the specified criteria are satisfied. If there are, a ChatMessageListUpdated event will be posted when more messages have been retrieved and are available to the client.

    Examples 

    // If the chat contains 1000 messages, and this function has not yet been
// called for the chat, it will retrieve messages 950-1000 and return
// them. If it is called again with the same parameters, it will retrieve
// messages 900-950 from the server and return messages 900-1000.
fetchChatMessages(chatId,{minFetchCount: 50,
                          syncMode: Messenger.SyncMode.NoSync})
    // If the chat contains 1000 messages, and this function has not yet
// been called for the chat, and messages 0-800 had previously been
// delivered to another client on which the messages were read, then it
// will retrieve messages 800-1000 and return them. If it is called again
// with the same parameters, it will retrieve messages 750-800 and return
// messages 750-1000.
fetchChatMessages(chatId, {minFetchCount: 50})

    Parameters

    Name Type Optional Description

    chatId

    string

     

    The chat for which messages are required.

    options

    Messenger.FetchChatMessagesOptions

    Yes

    The object defining the options to be applied while getting chat messages.

    Returns

    Promise containing any 

    Returns a promise of the new chat message list after pulling some of the messages. By default, the type of the message list is BBMEnterprise.Messenger.ChatMessage[] however this may be overridden by specifying a messageStorageFactory. The promise may be rejected if a failure occurs and additional messages could not be fetched.

    The total size of the 'content' in UTF-8 is subject to a cumulative size limit when combined with the size of the 'data' object (encoded as JSON in UTF-8) and the size of the 'thumbData' attachment. These three fields combined must not exceed 71680 bytes (70 KB).

    Object

    Yes

    Many 'tag' values have a corresponding object defined as a child of the 'data' object with a key equal to the 'tag'. All 'tags' with such a corresponding child of 'data' require that child to be present. Other tags have no such requirement. The 'data' object can also contain other key-value pairs independently from the 'tag'.

    The total size of the 'data' object (encoded as JSON in UTF-8) is subject to a cumulative size limit when combined with the size of the 'thumbData' attachment and the 'content' (also in UTF-8). These three fields combined must not exceed 71680 bytes (70 KB).

    (Uint8Array or Blob)

    Yes

    The raw bytes of a small file. The thumb data may contain no more than 56320 bytes (55 KB).

    The total size of the thumb data is subject to a cumulative size limit when combined with the size of the 'data' object (encoded as JSON in UTF-8) and the size of the 'content' (in UTF-8). These three fields combined must not exceed 71680 bytes (70 KB).

    thumbName

    string

    Yes

    A filename to associate with the thumbnail when a recipient stores it. On recipient mobile devices, the thumbnail data will be stored in a file with as similar a name to this as possible, subject to system and security rules. On recipient web clients, the exact name will be offered as a suggested filename. Because of filename rules on different platforms, only the following characters are guaranteed to be preserved:

    ChatStartOptions  Object

    An object to encapsulate the options used to start a chat via BBMEnterprise.Messenger#chatStart.

    Properties

    Name Type Optional Description

    subject

    string

     

    The subject of the chat. This may be empty. This field has a maximum size of 128 code points. Values larger than 128 code points will be truncated. If an existing 1:1 chat is chosen, and this subject differs, the existing chat's subject will be updated.

    isOneToOne

    boolean

    Yes

    When truthy, the chat is the singular 1:1 chat between the local user and the other party. The SDK ensures that only one such non-Defunct chat per remote party will exist at a time. Otherwise, the chat is a multi-party chat, even if there are fewer than two remote parties.

    invitees

    (Array of String or String)

    Yes

    The array of regIds of users being invited to join the chat once it has been created, or a single string to invite a single user. When isOneToOne is truthy, this list must contain a single regId, either in string form or as a single element array. Otherwise, it is optional or can be specified with 0 or more regId entries.

    When a single string is passed, a single Promise will be returned for the invite, rather than an array of promises.

    Creating a OneToOne chat with the regId of the local user is not supported. Creating a Conference where the regId of the local user is one of the invitees will result in the invite for this regId being rejected.

    invitePolicy

    Messenger.Chat.InvitePolicy

    Yes

    The policy that controls who may invite participants to the chat. This only applies when the 'isOneToOne' field is falsy. The default is 'ParticipantsOnly'

    data

    Object

    Yes

    This field contains opaque data managed by the app.

    This data can be read or written by any participant. Concurrent writes are resolved in a way that all participants will eventually see the same steady-state outcome.

    This field is suitable for shared chat metadata that changes infrequently such as the chat avatar URL, the identifiers of external resources associated with the chat, or a per-chat setting shared by all participants. Data that changes frequently should be sent to all participants via the BBMEnterprise.Messenger#chatMessageSend function by setting the 'data' property of the BBMEnterprise.Messenger.ChatMessageSendOptions.

    The 'data', encoded as JSON in UTF-8, must not exceed 71680 bytes (70 KB).

    When not defined, the chat data will default to an empty object.

    appMessageReceived  BBMEnterprise.Event.AppMessageReceived

    The event used to relay an app message that has been received. For more information on app messages, please refer to the BBMEnterprise.Messenger.AppMessage documentation.

    These messages are not retained by the SDK.

    chatAdded  BBMEnterprise.Event.ChatAdded

    The event used to indicate that the local user is now a participant in the given chat. Consumption of messages may proceed.

    This event is emitted whenever a new chat becomes known to the SDK. For example:

    • When BBMEnterprise.Messenger#chatStart completes successfully
    • When another endpoint for the user creates a chat
    • When another user invites this user to join a chat

    chatMessageAdded  BBMEnterprise.Event.ChatMessageAdded

    The event used to indicate that a chat message has been added to the identified chat. For example:

    • When a call to BBMEnterprise.Messenger#chatMessageSend has been made
    • When a new message has been received from another endpoint or participant in the chat

    chatMessageListUpdated  BBMEnterprise.Event.ChatMessageListUpdated

    The event used to indicate that a bulk change to the chat message list has been made. For example:

    • While the promise created by a call to BBMEnterprise.Messenger#fetchChatMessages is outstanding
    • When a call to BBMEnterprise.Messenger#chatShred is made
    • When another user in the chat shreds their messages

    chatMessageUpdated  BBMEnterprise.Event.ChatMessageUpdated

    The event used to indicate that a message within a chat has been updated. For example:

    • When a message's delivery or read state has changed
    • When a message is deleted by the local user
    • When a message is recalled
    • When a message is referenced by a new message in the chat

    chatRemoved  BBMEnterprise.Event.ChatRemoved

    The event used to indicate that the SDK has removed all data associated with the given chat. The chat details and messages are no longer accessible to the local user via the BBMEnterprise.Messenger interfaces. Only chats in which the user is no longer a participant will be removed.

    Once this event is emitted for a chat, no further events related to the chat will be emitted.

    chatTyping  BBMEnterprise.Event.ChatTyping

    The event used to indicate that a chat participant has started or stopped typing in the identified chat. The list of typing users for a chat may be retrieved by calling BBMEnterprise.Messenger#getTypingUsers.

    chatUpdated  BBMEnterprise.Event.ChatUpdated

    The event used to indicate that one or more properties of a chat, including its participant list has changed.

    This event will be emitted whenever a change to a chat's properties becomes known to the SDK. For example:

    • When BBMEnteprise.Messenger#chatUpdate completes successfully
    • When another endpoint for the user has updated the chat
    • When another user has update the chat
    • When the participant list of the chat has changed. This can occur when a participant joins or leaves the chat or when a participant's properties have been updated.

    userKeysChanged  BBMEnterprise.Event.UserKeysChanged

    The event indicating that the keys for a user have changed. This is not sent when the user gains their initial set of keys.
