BBMEnterprise

Create an instance of the BBMEnterprise object with the configuration necessary to interact with BlackBerry Secure Spark on a user's behalf. The returned instance provides the application access to the Spark SDK APIs for managing chats and media calls.

string

 

The identity of the user. This is determined by the authentication server.

configuration.description

string

 

A description of the endpoint. This should be an appropriate localized string based on what it can learn about the endpoint, for example "Windows PC" or "Firefox". This information about the endpoint will be displayed by other devices doing endpoint management. The maximum length is 2000 codepoints. Anything longer will be truncated.

configuration.getToken

BBMEnterprise#authTokenCallback

 

This function will be invoked to get a new authentication token. The implementation should call out to a server to retrieve a new authentication token. It should never return a cached token - the implementation will internally cache the token and will call out to this function only when there is no known token, or the known token is not valid.

configuration.getKeyProvider

BBMEnterprise#keyProviderCallback

 

When defined, the SDK will call this function when the application's implementation of the KeyProviderInterface is needed. Otherwise, the BlackBerry Key Management Service (KMS) will be used.

configuration.kmsArgonWasmUrl

string

Yes

The URL or relative path of the Argon2 WASM file. Only needed when running in KMS mode using the Argon2 WASM module in a location other than "environment_argon2.wasm".

configuration.filter

Array of string

Yes

Filter the chat list to only allow the local endpoint to interact with chats whose identifiers are among the specified set. This is the only mechanism by which a local endpoint may see a different set of chats than another endpoint belonging to the same user.

If unspecified, allow interactions with any chat for which the client has permission. An empty array is not the same as unfiltered, it results in not allowing interaction with any chats.

Note that to be invited to a new chat when using a filter, that chat must be added to the filter before the invite is received. The set may be modified by calling BBMEnterprise.Messenger#filterAdd and BBMEnterprise.Messenger#filterRemove.

When using a filter, creating a chat will result in the chat being added to the filter for the local endpoint only. Other endpoints belonging to the local user must call BBMEnterprise.Messenger#filterAdd to have access to the chat.

Any chats added to the filter during instantiation or subsequently using BBMEnterprise.Messenger#filterAdd are not affected by calls to BBMEnterprise#shutdown and BBMEnterprise#setup.

configuration.syncMode

Messenger.SyncMode

Yes

Set the default for what status needs to be seen in a message before ending an initial sync. The default is SyncToRead if unspecified.

configuration.logLevel

number

Yes

The amount of log output to produce. 0 produces the least logging, 3 produces the most. The default is 2 if unspecified.

configuration.nickname

string

Yes

A nickname for the endpoint. This information about the endpoint will be displayed by other devices doing endpoint management. It should be provided by the user, and may be anything they want. The maximum length is 2000 codepoints. Anything longer will be truncated.

configuration.messageStorageFactory

BBMEnterprise.MessageStorageFactory

Yes

A factory to construct the datastructures to store messages for a chat. This may be replaced in order to do things including associating data with each chat message, applying transforms such as filtering to the message list, or getting precise notifications of any changes to the message list. The default implementation will return a message list in the form of an immutable array.

retryServerRequests()

This function may be called at any time user activity resumes after an idle period. Any event that is considered to be a good representation of the user becoming interested in interacting with Spark again may be used to trigger this function, and thus will make a good event to use as a user-driven retry trigger. For example an app could call this function whenever the application goes from being 'not on the screen' or 'out of focus' to being 'on the screen' or 'in focus'. However, the app may also choose to tie this trigger to an explicit user action such as a 'Refresh' button.

The app does not need to 'rate limit' or otherwise filter calls to this function.

setupStart()

Start the SDK setup process using the event-based mechanism (versus the older call-back based mechanism of BBMEnterprise#setup). This is the only setup process that supports the BlackBerry Key Management Service (KMS).

As setup progresses BBMEnterprise#event:setupState events will be fired. The app should monitor these events and react accordingly.

If an error occurs that can't be retried automatically a BBMEnterprise#event:setupError event will be fired. The app can manually retry setup later.

The application may choose to call BBMEnterprise#shutdown at any time to abandon its attempt to setup the SDK. It may also take the additional step of instantiating a new SDK instance if it would like to change user identities.

Calling BBMEnterprise#shutdown will reset the BBMEnterprise#setupState to 'NotRequested' and BBMEnterprise#setupStart may be called again to re-initiate the event driven setup.

Fires
BBMEnterprise#event:setupState
BBMEnterprise#event:setupError
Throws

BBMEnterprise.Error.Error 

When setup is already in progress or completed.

shutdown()

Reset the SDK to its unsetup state. After calling this function, BBMEnterprise#setup may be called again to set up the Messenger. This does not allow changing the user identity of the Messenger. Changing identity requires creation of a new Messenger. If after creating a new Messenger, the old one will no longer be used, then this function should be called to stop the old one from performing network operations, and to make it collectible as garbage.

Fires
BBMEnterprise#event:setupState

syncStart()

This function is only used by apps that use the BlackBerry Key Management Service (KMS).

Called by the app when the BBMEnterprise#setupState is 'SyncRequired'.

If the BBMEnterprise#syncPasscodeState is 'New', then no sync data exists in KMS for this identity and it must create new sync data. The app must call BBMEnterprise#syncStart with a BBMEnterprise.SyncStartAction of 'New' and supply the 'passcode' that will be used to protect the new sync data in KMS.

If the BBMEnterprise#syncPasscodeState is 'Existing', then existing sync data was found in KMS, and the app can do one of the following:

Restore the existing sync data by calling BBMEnterprise#syncStart with a BBMEnterprise.SyncStartAction of 'Existing' and supply the existing 'passcode'.

Generate new sync data and delete the existing data by calling BBMEnterprise#syncStart with a BBMEnterprise.SyncStartAction of 'New' and a supply a new 'passcode' to protect the new sync data in KMS.

On successful sync with KMS the BBMEnterprise#setupState will advance to 'Success'.

On a sync failure the BBMEnterprise#setupState may transition back to 'SyncRequired' in which case the app can retry. On permenant errors a BBMEnterprise#event:setupError would be fired.

Events

registrationChanged  BBMEnterprise.Event.RegistrationChanged

The event used to indicate that the client's Spark registration succeeded or failed. This event also communicates the user's Spark identity details which are also available via BBMEnterprise#getRegistrationInfo.

This event may be emitted at any time after BBMEnterprise#setup has been called.

setupError  BBMEnterprise.Event.SetupError

The SDK setup failed with an error. This event can be emitted by the SDK any time after BBMEnterprise#setupStart has been called. This event may also be called after the BBMEnterprise#setupState has transitioned to 'Success' to indicate that the SDK is no longer setup.

When this event is emitted, the BBMEnterprise#setupState will be transitioned to 'NotRequested' and BBMEnterprise#shutdown will have been automatically called by the SDK.

setupState  BBMEnterprise.Event.SetupState

The event indicating a change in the setup state. This event can be emitted after BBMEnterprise#setupStart or BBMEnterprise#syncStart has been called.
